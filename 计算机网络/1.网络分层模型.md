# 网络分层模型

## OSI参考模型（7层）
Open System Interconnection，开放式系统互联，国际标准化组织（ISO）指定的理论模型，从下到上分为：
- 物理层
- 数据链路层
- 网络层
- 传输层
- 会话层
- 表示层
- 应用层

## TCP/IP模型（4层）
基于OSI建立的实际实现的模型，从下到上分为：
- 主机到网络层（比特）：对应OSI中的<物理层>和<数据链路层> 定义一些硬件的标准
- 网络层（数据帧）：对应OSI中的<网络层> IP协议，用来寻址
- 传输层（数据帧）：对应OSI中的<传输层> TCP，UDP协议
- 应用层（数据段）：对应OSI中的<会话层>，<表示层>，<应用层>

## TCP三次握手和四次挥手
### 三次握手
- 第一次握手：C(客户端)-> S(服务端)
- 第二次握手：S -> C
- 第三次握手：C -> S
<br/><br/>
第一次握手：C发送SYN=J，S收到SYN，保证C发送没问题 <br/>
第二次握手：S发送SYN=K和ACK=J+1，C接到数据保证S接收能力以及自己发送能力没问题 <br/>
第三次握手：C发送ACK=K+1,S收到后保证C接收能力没问题 <br/>
----
### 四次挥手
- 第一次挥手：C(客户端)-> S(服务端)
- 第二次挥手：S -> C
- 第三次挥手：S -> C
- 第四次挥手：C -> S
<br/><br/>
第一次挥手：C发送FIN=X,告诉S今后不会再发送数据了，S收到后不会写入数据了 <br/>
第二次挥手：S仍有一些数据需要发送，同时发送ACK=X+1 <br/>
第三次挥手：S发送FIN=Y,告诉C今后不会再给你发数据了 <br/>
第四次挥手：C发送Ack，连接结束<br/>
> 挥手服务端发送两次是为了允许客户端主动断开连接后仍可发送一些数据
